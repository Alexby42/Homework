Для графического отображения игры используется стандартная библиотека Tkinter, а также библиотека random, для генерации случайных чисел.
В начале кода создается окно будущей игры и объявляются переменные.
game_run - содержит булевые переменные, для отслеживания, чтобы не было возможности ходить, когда установлен победитель.
field - поле игры, список в который схоаняются кнопки. При осуществлении хода на кнопке появляется символ (sbl) Х или О.
count - переменная призванная отслеживать Х на поле. При выставлении пятого Х фиксируется "ничья", если нет победителя.
Функция new_game вызывается при нажатии кнопки "Новая игра". Удаляются все значения с поля и глобальные переменные game_run и count
устанавливаются в начальные значения.
Функция click вызывается при нажатии кнопки поля и ставится Х, поскольку первый ход за ним. Переменная count начинает вести отсчет X.
Потом вступает функция check_win для проверки условия победы. Если победителя нет, вступает функция computer_step и снова идет проверка условия
победы.
Функция check_win проверяет выигрыш. Происходит перебор комбинации полей в виде линий и вызывается функция check_line.
Переменная sbl является символом Х или О. Если задан О, идет проверка победы компьютера. Если фиксируется выигрыш линия кнопок меняет цвет и в
переменную game_run записывается значение False.
Ходы компьютера определяются функцией computer_step. Если есть вариант закрыть линию, происходит победа копьютера. Если есть вариант победы
противника, компьтер закрывает оставшуюся для победы ячейку. Если нет форс-мажорных обстоятельств, посредством цикла while определяется
случайное число и занимется свободная ячейка.
Элементы графики размещаются с помощью упаковщика grid из библиотеки Tkinter. При помощи цикла добавляются кнопки поля методом append, так как
в данном случае используется список.
